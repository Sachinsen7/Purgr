# Security Policy

## Supported Versions

Purgr is still in early development. Security fixes are applied on the latest
main branch first.

## Reporting a Vulnerability

Please do not open a public GitHub issue for security-sensitive reports.

Instead, send details to:

- `security@purgr.dev`

Please include:

- A clear description of the issue
- Steps to reproduce
- The potential impact
- Any proof-of-concept details needed to validate the report

We will acknowledge receipt as quickly as possible and work with you on
validation, impact assessment, and remediation.

## Scope

Security reports are especially helpful for:

- Unsafe file deletion behavior
- Sidecar or IPC privilege issues
- Path traversal or arbitrary file access
- Secrets exposure
- Supply-chain or build integrity issues
- Vulnerabilities in packaged desktop releases
